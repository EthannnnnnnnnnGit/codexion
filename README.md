## Description
```
Codexion is a 42 project made to understand multi-threading. It's a simulation of threads doing tasks that are in order compiling, debugging and refactoring.  
All of the tasks are dones in nb_of_compiles times and have their own 
The project should have a thread representing a monitor to manage other threads burnout.  
During this project, multiple thread functions are used to learn the behaviour of thread, mutex and conditionnal waiting.



```

## Instructions

To compile the project, you have to run make, it will create the binary executable as well of objects and dependencies in .objs folder.

To run the project, use the executable with ./codexion nb_of_coders time_of_burnout time_to_compile time_to_debug time_to_refactor nb_of_compiles dongle_cooldown scheduler

To erase objects and dependencies, run make clean.
To erase both binary annd objects, run make fclean.

To remake all of the project, run make re.
